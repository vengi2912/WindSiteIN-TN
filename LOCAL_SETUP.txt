OpenGridPK — running locally
============================

This document walks through the full local setup on Windows, macOS, or Linux.
The project has two halves:

  1. A Python build pipeline that fetches data from OpenStreetMap and writes
     GeoJSON files into site/data/.

  2. A static HTML/CSS/JS site that loads those GeoJSON files and renders the
     map with MapLibre GL JS.

Both halves run from the same repository. There is no backend, no database,
and nothing to deploy beyond the static files.

Prerequisites
-------------

  - Git
  - Python 3.12 or newer (3.14 is fine; tested on both)
  - A modern web browser

That's it. No Node, no Docker, no cloud account.

To verify what you have:

    git --version
    python --version

Step 1 — Clone the repository
-----------------------------

    git clone https://github.com/<your-username>/opengridpk.git
    cd opengridpk

Step 2 — Create the Python virtual environment
----------------------------------------------

From the repository root:

    cd pipeline
    python -m venv .venv

Activate the virtual environment.

    Windows (PowerShell):
        .\.venv\Scripts\Activate.ps1

    Windows (cmd.exe):
        .\.venv\Scripts\activate.bat

    macOS / Linux:
        source .venv/bin/activate

Step 3 — Install the pipeline package
-------------------------------------

Still inside pipeline/, with the venv active:

    python -m pip install --upgrade pip
    python -m pip install -e ".[dev]"

This installs the build pipeline in editable mode along with the test
dependencies (pytest, ruff). Total install size is small — a few MB.

Step 4 — Run the tests (optional but recommended)
-------------------------------------------------

    python -m pytest

You should see roughly 48 tests pass in under two seconds. If anything
fails, stop and report the error; downstream steps depend on the modules
the tests cover.

Step 5 — Build the data
-----------------------

    python -m pipeline.refresh

This issues several queries against the public Overpass API and writes
fresh GeoJSON files into ../site/data/. The first run takes a couple of
minutes; subsequent runs are cached and finish in seconds (cache lives in
pipeline/.cache/, which is gitignored).

Expected output (numbers will drift as OSM evolves):

    [refresh] 500kv: 229 lines, 7468 km
    [refresh] hvdc: 5 lines, 887 km
    [refresh] 220kv: 319 lines, 6855 km
    [refresh] 132kv: 1432 lines, 20061 km
    [refresh] 66kv: 125 lines, 2361 km
    [refresh] substations: 1177 features
    [refresh] generation: 231 features
    [refresh] done.

Step 6 — Serve the site
-----------------------

From pipeline/ (still in the active venv) or the repo root:

    python -m http.server 8000 --directory ../site

Or, equivalently, from inside site/:

    python -m http.server 8000

Then open in your browser:

    http://localhost:8000

You should see the OpenGridPK header, the legend on the right with five
voltage classes plus substations and generation toggles, and the country
view of Pakistan with all the lines drawn.

Step 7 — Make changes
---------------------

Common edits:

  - Change voltage colours / labels / zoom thresholds:
        data/reference/voltage_classes.json
    (Used by both the pipeline and the site. Run `make refresh` after
     editing so the file is copied into site/data/.)

  - Add a hand-curated correction:
        data/overrides/voltage_corrections.json
        data/overrides/name_corrections.json
        data/overrides/add_lines.geojson
    See data/overrides/README.md for formats.

  - Adjust the site's appearance:
        site/style.css
        site/app.js
        site/index.html

  - Adjust the pipeline:
        pipeline/pipeline/*.py
    Tests live next to them in pipeline/tests/.

After any pipeline edit, re-run `python -m pipeline.refresh` to regenerate
the GeoJSON files.

Optional — Makefile shortcuts
-----------------------------

If you have GNU make installed (Git Bash on Windows includes it):

    make install    # creates .venv + installs deps (one time)
    make refresh    # rebuilds GeoJSON
    make serve      # starts the static server
    make test       # runs pytest
    make lint       # runs ruff

Run these from inside the pipeline/ directory.

Troubleshooting
---------------

* "Overpass timeout" or "All Overpass endpoints failed":
  The public Overpass mirrors throttle aggressively. Wait a few minutes
  and rerun. The pipeline already retries against three mirrors with
  exponential backoff.

* The browser shows the legend but no lines:
  Open the browser dev tools, check the Network tab. Every
  data/lines_*.geojson should return 200. If any return 404, you haven't
  run the refresh yet — go back to Step 5.

* "Module not found: pipeline.xxx":
  Either the venv isn't active, or the package wasn't installed in
  editable mode. Re-run Step 3.

That's all. Once you've gone through this once, day-to-day work is just:

    cd pipeline
    .\.venv\Scripts\Activate.ps1   (or the equivalent for your shell)
    python -m pipeline.refresh
    python -m http.server 8000 --directory ../site
